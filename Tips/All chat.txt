GitHub Copilot: Short answer: you don't strictly need a virtual environment, but it's strongly recommended â€” especially when you install Python packages (like a tool to convert SVG â†’ PDF). A venv keeps dependencies isolated, reproducible, and safe from system Python changes so your project stays portable and easier to share.

Why use a venv (concise):
- Isolation: packages you install (cairosvg, etc.) go into the venv, not the system Python. No version conflicts with other projects.  
- Reproducibility: you can export a `requirements.txt` and anyone can recreate the same environment.  
- Safety & permissions: no sudo installs, no accidental upgrades to system packages.  
- Multiple Python versions: different projects can use different interpreters/versions.  
- Clean removal: delete the venv to remove all installed packages.
